
Development Rules

Before writing any code:

1. Read the existing project structure.
2. Reuse every existing Blade component whenever possible.
3. Never duplicate components.
4. Never rewrite unrelated files.
5. Never reinstall packages that already exist.
6. Preserve project architecture.
7. Explain the implementation plan before making changes.
8. Implement one feature at a time.
9. Stop after completing the requested sprint.
10. Wait for the next instruction.
